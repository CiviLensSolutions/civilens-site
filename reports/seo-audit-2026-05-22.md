## Critical Issues — Fix Immediately

### C1. No robots.txt (+ noindex conflict risk)
`/robots.txt` returns 404. The 404 page renders with `<meta name="robots" content="noindex">` — crawlers hit a noindex signal when fetching crawl directives.

**Conflict note:** Technical agent confirmed homepage itself is NOT noindexed (`x-nextjs-prerender: 1` confirmed). Sitemap agent detected possible noindex on the homepage. Manually verify: `curl -s https://www.civilens.solutions | grep -i noindex`

**Fix:** Create `src/app/robots.ts`:
```ts
import type { MetadataRoute } from "next";
export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: "*", allow: "/" },
    sitemap: "https://www.civilens.solutions/sitemap.xml",
  };
}
```

### C2. No sitemap.xml
Both `/sitemap.xml` and `/sitemap_index.xml` return 404. No `sitemap.ts` in `src/app/`.

**Fix:** Create `src/app/sitemap.ts`:
```ts
import type { MetadataRoute } from "next";
export default function sitemap(): MetadataRoute.Sitemap {
  return [{ url: "https://www.civilens.solutions", lastModified: new Date() }];
}
```

### C3. Zero schema markup
No JSON-LD, no Microdata, no RDFa anywhere in the codebase. No knowledge panel. No local pack eligibility. AI systems cannot cite the business.

**Fix:** Add a `<script type="application/ld+json">` block in `layout.tsx` with a `@graph` array containing:
- `ProfessionalService` + `Organization` (name, address, email, telephone, geo, areaServed, founder)
- `WebSite`
- `Person` (Shea Scott — name, jobTitle, worksFor, knowsAbout, description)
- 3x `Service` (AI Integration, Process Automation, Web & Software)

Full JSON-LD blocks were generated by the schema subagent — use those directly.

### C4. No phone number — GBP blocker
No phone number anywhere on the site. Blocks GBP verification, citation submissions, NAP completeness, and `telephone` schema property.

**Fix:** Add a business phone (Google Voice acceptable) to footer and schema before any GBP or citation work.

### C5. AI search readiness: 31/100 — key content client-rendered
`Capabilities` cards and `CalEmbed` pitch column are client-rendered — invisible to AI crawlers. No `llms.txt`. No robots.txt to explicitly allow AI crawlers.

**Fix:** Convert `Capabilities` to server component (hover state only needs a thin client wrapper). Split `CalEmbed` so static pitch copy renders server-side; only the widget mount div stays client-rendered.

## High Priority — Fix Within 1 Week

### H1. Bare domain redirect is HTTP 307 (temporary)
`civilens.solutions` → `www.civilens.solutions` via 307. Temporary redirects do not pass full PageRank.

**Fix:** Vercel dashboard → Domains → change redirect type to permanent (308).

### H2. No canonical tag
No `<link rel="canonical">` on homepage. With a 307 between apex and www, Google may treat both as separate URLs.

**Fix:** In `layout.tsx`:
```ts
export const metadata: Metadata = {
  alternates: { canonical: "https://www.civilens.solutions" },
};
```

### H3. No meta description rendering
Verify with `curl -s https://www.civilens.solutions | grep -i "meta name.*description"`. Google may be auto-generating the snippet.

### H4. Missing security headers
No `Content-Security-Policy`, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`, or `Permissions-Policy`.

**Fix:** Add to `next.config.ts` headers array. CSP must allowlist `https://cal.com` and `https://app.cal.com` in `frame-src` for the embed.

### H5. No LCP preconnect for Cal.com
Cal embed loads via `useEffect` with no preconnect hint — cascading third-party load in critical path.

**Fix:** Add `<link rel="preconnect" href="https://app.cal.com" />` to `<head>` in `layout.tsx`.

### H6. No keyword in any heading
"AI consulting for government," "government AI integration," "Denton TX" — none appear in H1–H3. H1 is screen-reader only and uses brand language ("Civic Intelligence") not search language.

**Fix:** Revise first visible H2 to include a searchable phrase, e.g. _"AI Consulting for Government Agencies and Nonprofits"_

### H7. No GBP listing confirmed
Zero GBP signal on site. No Maps embed, Place ID, or directions link.

**Fix:** Create and verify GBP. Primary category: `IT consultant`. Secondary: `Business management consultant`, `Software company`. Configure as Hybrid (address shown + DFW service area).

### H8. No privacy policy — TDPSA legal exposure
Cal.com collects names and emails. No privacy policy exists. Violates Google quality guidelines and **Texas Data Privacy and Security Act (effective 2024)**.

**Fix:** Add `/privacy` page and link in footer.

## Medium Priority — Fix Within 1 Month

### M1. Page-type mismatch (SXO: 34/100)
Single-page homepage vs. multi-page authority sites dominating target SERPs. One URL cannot rank for multiple queries.

**Minimum viable expansion:**
- `/services/ai-integration` (1,000–1,500 words)
- `/services/process-automation`
- `/services/web-software`
- `/about` (verifiable project history)
- `/contact`

### M2. Content too thin (~355–600 words)
Homepage minimum for this niche: 500 words. Missing: client social proof, compliance language (Section 508, FISMA), project specifics, FAQ, methodology.

### M3. Name inconsistency
`CiviLens` (brand), `Civilens LLC` (copyright) — never mix in structured data. Use `CiviLens` for brand, `Civilens LLC` only in legal/copyright contexts.

### M4. No Tier-1 citations
Yelp, Bing Places, Apple Business Connect, LinkedIn Company Page — all unconfirmed. **Clutch.co is highest B2B priority** — government procurement teams actively search Clutch for vendors.

### M5. No government procurement credentials on site
No SAM.gov UEI, no NAICS codes, no SDVOSB designation, no downloadable capability statement. Procurement officers cannot qualify the vendor without a call — they won't make the call.

### M6. E-E-A-T gaps
- Authoritativeness: 7/25 — no external citations, no press, no professional associations
- Experience: 14/20 — no named projects, agencies, or outcomes
- Expertise: 15/25 — no compliance vocabulary (Section 508, FedRAMP, CMMC)
