## Findings

### SHIELD-001 — PostCSS XSS via Unescaped `</style>`
- **Severity:** MEDIUM
- **CWE:** CWE-79 (Improper Neutralization of Input During Web Page Generation)
- **OWASP:** A06:2021 — Vulnerable and Outdated Components
- **Package:** postcss < 8.5.10
- **Impact:** If postcss processes any user-controlled or external CSS, an unescaped `</style>` tag in output could break out of the style block and inject arbitrary HTML/JS.
- **Fix:** Update Next.js to latest — it pins a patched postcss version transitively.
- **Compliance:** SOC 2 CC6.1 | PCI-DSS 6.3.2

### SHIELD-002 — Next.js ships vulnerable postcss range
- **Severity:** MEDIUM
- **CWE:** (transitive via postcss)
- **OWASP:** A06:2021 — Vulnerable and Outdated Components
- **Package:** next (9.3.4-canary.0 – 16.3.0-canary.5 range affected)
- **Impact:** Root cause is the postcss dependency. Next.js 15.x with updated lockfile resolves this.
- **Fix:** `npm update next` or bump to latest stable.
- **Compliance:** SOC 2 CC6.1 | PCI-DSS 6.3.2
